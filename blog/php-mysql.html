PHP & MySQL: Best practices voor beginners

De belangrijkste lessen die ik heb geleerd over veilig en efficiënt werken met databases.

Dit artikel is beschikbaar in het Nederlands.

Tijdens mijn opleiding en projecten heb ik veel geleerd over PHP en MySQL. Sommige lessen kwamen uit documentatie, andere uit fouten die ik zelf maakte. In dit artikel deel ik de belangrijkste best practices die elke beginner zou moeten kennen.

1. Voorkom SQL Injection

Dit is de belangrijkste regel. Nooit, maar dan ook nooit, user input direct in je SQL query plaatsen:

FOUT - Doe dit NOOIT 
$query = "SELECT * FROM users WHERE id = " . $_GET['id'];
$result = mysqli_query($conn, $query);
GOED - Prepared Statements 
$stmt = $conn->prepare("SELECT * FROM users WHERE id = ?");
$stmt->bind_param("i", $_GET['id']);
$stmt->execute();
$result = $stmt->get_result();

Tip: Gebruik PDO of MySQLi met prepared statements. Altijd. Geen uitzonderingen.

2. Hash Wachtwoorden Correct

Sla wachtwoorden nooit op als plain text. Gebruik altijd PHP's ingebouwde functies: 

// Bij registratie
$hashedPassword = password_hash($password, PASSWORD_DEFAULT);

// Bij login
if (password_verify($inputPassword, $hashedPassword)) {
    // Wachtwoord correct
}

PASSWORD_DEFAULT gebruikt automatisch het beste beschikbare algoritme (momenteel bcrypt).

3. Error Handling

Toon nooit database errors aan gebruikers in productie. Log ze in plaats daarvan: 

try {
    $stmt = $pdo->prepare($sql);
    $stmt->execute($params);
} catch (PDOException $e) {
    // Log de error
    error_log("Database error: " . $e->getMessage());

    // Toon gebruikersvriendelijke melding
    echo "Er ging iets mis. Probeer het later opnieuw.";
}

4. Database Normalisatie

Een goed ontworpen database bespaart je later veel problemen. De basis regels:

1NF - Eerste Normaalvorm

Elke kolom bevat atomaire (ondeelbare) waarden. Geen lijsten in één veld.

2NF - Tweede Normaalvorm

Alle niet-sleutel kolommen zijn volledig afhankelijk van de primaire sleutel.

3NF - Derde Normaalvorm

Geen transitieve afhankelijkheden tussen niet-sleutel kolommen.

Foreign Keys

Gebruik foreign keys voor relaties en data integriteit.

Checklist voor je project

  • Prepared statements voor alle queries
  • Wachtwoorden gehasht met password_hash()
  • Error handling met try-catch
  • Input validatie en sanitization
  • Database connection in apart config bestand
