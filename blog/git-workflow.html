Git workflow tips voor solo developers

Ook als je alleen werkt is een goede Git workflow essentieel voor je productiviteit en code kwaliteit.

Dit artikel is beschikbaar in het Nederlands.

"Ik werk alleen, waarom zou ik branches gebruiken?" Dit dacht ik ook in het begin. Maar na een paar keer code kwijtraken en uren debuggen, heb ik geleerd dat een goede Git workflow ook voor solo developers onmisbaar is.

1. Schrijf duidelijke commit messages

Je toekomstige zelf zal je dankbaar zijn. Een goede commit message legt uit wat en waarom:

Niet zo
"fix"
"update"
"changes"
"asdfasdf"
Beter
"fix: login form validation"
"feat: add dark mode toggle"
"refactor: simplify nav logic"

Tip: Gebruik prefixes als feat:, fix:, refactor: voor consistentie.

2. Gebruik branches (ook als solo dev!)

Branches geven je de vrijheid om te experimenteren zonder je werkende code te breken: 

# Nieuwe feature starten
git checkout -b feature/dark-mode

# Werk aan je feature...
git add .
git commit -m "feat: implement dark mode toggle"

# Terug naar main en mergen
git checkout main
git merge feature/dark-mode

# Branch opruimen
git branch -d feature/dark-mode
feature/naam fix/bug-naam experiment/idee

3. Vergeet je .gitignore niet

Voorkom dat je per ongeluk gevoelige data of onnodige bestanden commit: 

# Dependencies
node_modules/
vendor/

# Environment variables (BELANGRIJK!)
.env
.env.local
config.php

# IDE settings
.vscode/
.idea/

# Build output
dist/
build/

# OS files
.DS_Store
Thumbs.db

4. Push regelmatig naar remote

GitHub/GitLab is niet alleen voor samenwerking - het is ook je backup:

  • Push aan het einde van elke werkdag
  • Push na elke afgeronde feature
  • Gebruik private repos voor gevoelige projecten

Handige commando's

git status

Bekijk huidige status

git log --oneline

Compacte commit history

git diff

Bekijk unstaged changes

git stash

Tijdelijk changes opslaan
